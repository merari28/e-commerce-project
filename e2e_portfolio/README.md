# Merari Store — E2E Screens & Portfolio Gallery

End-to-end captures of **every screen** of the e-commerce app (Django 6),
generated by driving a real Chromium browser through the full user journey:
browse → search → register/login → cart → checkout → PayPal → order complete,
plus the Django admin panel.

The images in [`screenshots/`](./screenshots) are full-page, 2× device scale
(retina) and ready to drop into a portfolio.

## How it was generated

1. Local server runs on a free port with a self-contained SQLite database
   seeded from `data.json` (11 products, 5 categories, 9 users, 10 orders,
   payments and a review) plus generated product/category images.
2. [`capture_screens.py`](./capture_screens.py) uses Playwright to visit each
   route, perform the real flows (login, add-to-cart, checkout, simulated
   PayPal approval) and save a screenshot per screen.

### Re-run it yourself

```bash
# 1. start the server (from the project root)
USE_SQLITE=1 venv/Scripts/python.exe manage.py runserver 127.0.0.1:8010 --noreload

# 2. in another shell, capture all screens
USE_SQLITE=1 venv/Scripts/python.exe e2e_portfolio/capture_screens.py
```
